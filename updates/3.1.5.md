# What has been updated?

## Changelog - (Added, Fixed, Removed):

1. Fixed - Improved the symbols usage on the telegram and parcel messages, they were disabled because they were causing issues, but now everything works thanks to @Twacker.

2. Fixed an issue with job telegrams, they were broken due to the last updates for supporting multiple jobs on the same job section.

3. Added new feature for selecting through the configuration file the sender username display in case some servers do not wish to display sender fullname but only last, first or just telegram id.

## Notes

1. This is the new configuration option on `Config.Jobs`, a new option parameter has been added (`TelegramId`).

```lua
-- @param PermittedGrades : The permitted grades to be able to open the job telegrams and read, send or reply.
-- @param SendTelegramToEveryone : Set to true if you want the specified job to 
-- be able to use -1 for sending telegrams to everyone.
-- @param TelegramId : The telegram id for the specified job(s) - (version 3.1.5)
Config.Jobs = { -- 3.0.8
    { Label = 'Police Department',  PermittedJobs = {'police'}, PermittedGrades = { { 3, 4 } }, SendTelegramToEveryone = true, TelegramId = 'police'  },
    { Label = 'Medical Department', PermittedJobs = {'medic'}, PermittedGrades = { { 1 }  },    SendTelegramToEveryone = true, TelegramId = 'medical' },
}
```

2. Below `Config.Jobs` add the new configuration option for the new feature.

```lua
-- __Display Types__
-- 1. FULLNAME : Displays first and lastname of the sender.
-- 2. FIRSTNAME : Displays only the first name of the sender.
-- 3. LASTNAME : Displays only the lastname of the sender.
-- 4. TELEGRAM_ID : Displays only the telegram id of the sender.

-- (!) All the changes will be functional only on the new telegrams or parcels,
-- existing ones will not be edited, they will remain the same.
Config.DisplaySenderNameType = "FULLNAME" -- version 3.1.5

```

## How to update?

- You can easily update the modified files by redownloading from the **Cfx.re Portal** and copy-paste on the mentioned directory file paths.
- If new options have been implemented to the configuration file or edited, **DO NOT** copy-paste the new configuration file, otherwise you will get the latest configuration but you will lose all the changes you have made. The proper way updating a configuration file is by finding the new configuration options or the modified ones and do it **manually**.

*Cfx.re Portal **DOES NOT** force-update your configuration file or any files, if it did, you would always lose all the configuration changes you made or would totally break the script, all files are updated or edited manually.*
