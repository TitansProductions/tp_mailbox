# What has been updated?

## Changelog - (Added, Fixed, Removed):

1. Added a new option for **tp_mailbox_mailman** upcoming script support and features related to that script.
2. Added stamps to be able to be used as items as an alternative method when using **tp_mailbox_mailman**.
3. Added a new system that allows you to get a telegram as a document and read it, including the nearest players (optional).
4. Added a new system that blocks specific keys from being triggered when having NUI open - this prevents horse or wagon whistling.
5. Modified the telegram id selection on Telegrams and Employement sections, before, you had to select the title to open a telegram or a parcel, now you just select anywhere - hovering opacity effect has also been modified on this change.
6. Added new configuration option to disable or enable telegrams list icons (open parcel, closed parcel, open telegram, close telegram)
7. Increased max pages for telegrams up to 40 pages displayed from 20 pages that was the maximum.
8. Added new searching system to search telegrams through a title, a player telegram id or player first and last name.
9. Added notification sound configuration option through `interact-sound` script for those who are interested to receive sound notification when receiving a parcel or telegram.
10. Added (1) more extra function on `client/tp-client_escrow_ignore.lua` which was missing and should be from previous updated related to target for mailbox objects.

In case you want to keep your `client/tp-client_escrow_ignore.lua` file, add this manually:

```lua
-----------------------------------------------------------
--[[ Functions - version 1.3.8 ]]--
-----------------------------------------------------------
-- The following task is made to prevent opening specific keys while nui is active.
Citizen.CreateThread(function()
    while true do 
        
        local sleep  = 1500
    
        local player = PlayerPedId()
        local isDead = IsEntityDead(player)

        if isDead or not ClientData.Loaded then
            goto END
        end

        if ClientData.IsBusy then 
            sleep = 0

            DisableControlAction(0, Config.HorseWhistleKey, true) -- Jump
			DisableControlAction(0, Config.WagonWhistleKey, true) -- Jump

        end

        ::END:: 
        Wait(sleep)

    end

end)
```

## How to update?

- You can easily update the modified files by redownloading from the **Cfx.re Portal** and copy-paste on the mentioned directory file paths.
- If new options have been implemented to the configuration file or edited, **DO NOT** copy-paste the new configuration file, otherwise you will get the latest configuration but you will lose all the changes you have made. The proper way updating a configuration file is by finding the new configuration options or the modified ones and do it **manually**.

*Cfx.re Portal **DOES NOT** force-update your configuration file or any files, if it did, you would always lose all the configuration changes you made or would totally break the script, all files are updated or edited manually.*
